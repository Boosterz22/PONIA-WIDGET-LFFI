Yes — l’erreur vient de LI.FI : le paramètre fromAddress est requis pour les quotes (même si tu quotes le token natif). On n’a pas encore de wallet branché dans la démo, donc on doit fournir une adresse EVM “dummy” (valide) côté requête. Tu peux aussi mettre toAddress au même dummy.

Patch ultra-simple

Ouvre app.js et fais ces deux changements seulement :
	1.	Ajoute une adresse de démo tout en haut, avec les autres constantes :

const DEMO_EVM_ADDRESS = '0x0000000000000000000000000000000000000001';

	2.	Remplace ta fonction fetchQuote par celle-ci (elle envoie fromAddress et toAddress + un tag integrator) :

async function fetchQuote({ fromChain, toChain, amountWei, fromAddress = DEMO_EVM_ADDRESS, toAddress = DEMO_EVM_ADDRESS }) {
  const params = new URLSearchParams({
    fromChain,
    toChain,
    fromToken: '0x0000000000000000000000000000000000000000', // natif
    toToken:   '0x0000000000000000000000000000000000000000', // natif
    fromAmount: amountWei,
    fromAddress,
    toAddress,
    integrator: 'ponia-demo'
  });

  const res = await fetch(`https://li.quest/v1/quote?${params.toString()}`);
  if (!res.ok) {
    const txt = await res.text();
    throw new Error(`LI.FI error ${res.status}: ${txt}`);
  }
  return res.json();
}

Rien d’autre à changer : l’appel dans onQuote() marchera tel quel, car la fonction met déjà des valeurs par défaut.

Relance et teste : Ethereum → Polygon, saisis un montant (ex. 0.1) puis “Get quote”. Tu dois voir la route, l’outil (ex. stargate, hop, …), l’ETA et l’output estimé.

Notes rapides
	•	Cette démo reste EVM-only (ETH/MATIC/BNB…). Pour Solana ou d’autres non-EVM, on ajoutera un mapping d’adresses et un switch d’API (LI.FI le gère, mais les paramètres d’adresse changent).
	•	Quand tu brancheras un wallet réel, on remplacera DEMO_EVM_ADDRESS par l’adresse signée de l’utilisateur, et on pourra enchaîner l’exécution de la route (pas seulement la quote).

Si tu veux, je te fais la variante USDC multi-chaînes (avec adresses de token correctes) dans le même style.
